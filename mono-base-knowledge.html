How Mono Base works

A browsable library of the "basic" knowledge that's easy to have missed — deepest on vehicles, design & materials, history, and sports.

Library

Cards per section, searchable across everything. KNOWN marks a card learned (tracked in the progress bar). → MIND saves the card as a note in Mono Mind, where it enters the 1/3/7/21-day review cycle — that's your "remember this later" button.

Faces

Every name gets a face chip. Photos load live from Wikipedia (needs internet and a real browser — on the hosted site they just work). Offline you get initials plus the mnemonic hook, which is honestly the part that makes names stick. Tap any person for the full card and a link to read more.

Face quiz

Shows a loaded face, you pick the name from three. Pure name–face encoding reps. Browse the library online once first so faces are cached.

Send to Mind — the fine print

Both apps must run from the same place for the handoff to work: the hosted tools site (best), or the same browser on your PC. It writes the note locally and bumps the sync clock so Mind pushes it to GitHub next time it opens.

GitHub sync

Same repo and token as the other apps; this one writes data/mono-base.json. Token stays on-device, never uploaded or exported.
